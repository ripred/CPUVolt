# Support

Use the repository documentation first:

- [README](README.md)
- [Examples](examples/)

Open a GitHub issue when you have a reproducible bug, documentation problem, example problem, or feature request.

For support requests, include:

- Project version, release, or commit.
- Board and Arduino core, when hardware is involved.
- Relevant libraries and versions.
- The sketch, example, or generated output involved.
- A small sketch, command, or exported artifact that reproduces the problem.
- Exact compiler output, serial output, browser console message, or observed behavior.

General electronics debugging and project-specific wiring are easier to help with when the report includes a schematic, pin list, and the smallest sketch that still shows the issue.
