# Code of Conduct

This project space should stay useful for people building, testing, documenting, and debugging Arduino and embedded work.

## Expected Behavior

- Be direct, specific, and respectful.
- Keep issue and review comments focused on the code, docs, examples, hardware setup, or generated output.
- Assume embedded constraints are real: memory, timing, toolchain age, board quirks, and wiring details often matter.
- Point out risks and tradeoffs clearly without making the discussion personal.
- Help keep examples, bug reports, and pull requests reproducible.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or sustained hostile behavior.
- Dismissive comments about someone else's skill level, hardware, platform, or tool choice.
- Spam, off-topic promotion, or repeated low-effort comments after maintainers ask for clarification.
- Publishing private contact information, credentials, vulnerability details, or other sensitive material.

## Enforcement

Maintainers may edit or remove comments, close issues, block users, or restrict participation when behavior makes the project harder to maintain or unsafe for contributors.

If a conduct problem happens, open a concise issue if it is safe to do so. If it involves sensitive information, use the security reporting guidance instead of posting details publicly.
